## 1.11. Thiết kế cách tiến hành Test

### Tổng quan chiến lược kiểm thử

Hệ thống được kiểm thử theo 3 cấp độ: **Unit Test** (kiểm tra từng thành phần độc lập), **Integration Test** (kiểm tra tương tác giữa các module), và **Manual Test** (kiểm tra giao diện và luồng nghiệp vụ thực tế). Công cụ chính sử dụng PHPUnit (tích hợp sẵn với Laravel) cho phía server và kiểm thử thủ công trình duyệt cho phía giao diện.

### 1.11.2. Integration Test — Kiểm thử tích hợp

#### a) API Raspberry Pi ↔ Laravel

**Công cụ:** PHPUnit + Laravel HTTP Test / Postman

| STT | Endpoint | Kịch bản | Kết quả mong đợi |
|---|---|---|---|
| 1 | `POST /api/auth/device` | Token hợp lệ | HTTP 200, trả về user device info |
| 2 | `POST /api/auth/device` | Token sai | HTTP 401 |
| 3 | `GET /api/encodings` | Không có tham số `updated_since` | Trả toàn bộ encodings |
| 4 | `GET /api/encodings?updated_since=...` | Có timestamp | Chỉ trả encodings mới hơn timestamp |
| 5 | `POST /api/attendance` | Dữ liệu hợp lệ, nhân viên có ca | HTTP 201, bản ghi được tạo với status đúng |
| 6 | `POST /api/attendance` | Cùng user_id trong 5 phút (cooldown) | HTTP 200, bỏ qua (không tạo trùng) |
| 7 | `POST /api/attendance/batch` | Mảng nhiều bản ghi offline | HTTP 200, xử lý trùng theo (user_id, work_date, shift_schedule_id) |
| 8 | `POST /api/device/ping` | Token hợp lệ | HTTP 200, cập nhật last_ping và status = online |

#### b) Queue Job — Face Encoding

| STT | Kịch bản | Kết quả mong đợi |
|---|---|---|
| 1 | Admin upload ảnh khuôn mặt hợp lệ | Job được dispatch, chạy Python script, lưu encoding 128D vào DB |
| 2 | Upload ảnh không có khuôn mặt | Job chạy, không lưu encoding, đánh dấu lỗi |
| 3 | File ảnh sai định dạng | Validation từ chối trước khi dispatch |

#### c) Luồng Offline Sync

| STT | Kịch bản | Kết quả mong đợi |
|---|---|---|
| 1 | Batch sync không trùng | Tất cả bản ghi được lưu |
| 2 | Batch sync có bản ghi trùng (user_id + work_date + shift) | Bản ghi cũ được giữ, không tạo trùng |
| 3 | Batch sync bản ghi với timestamp cũ hơn bản ghi hiện tại | Hệ thống ưu tiên bản ghi mới hơn |

### 1.11.5. Công cụ và môi trường kiểm thử

| Loại test | Công cụ |
|---|---|
| Unit / Integration (PHP) | PHPUnit 12 (tích hợp Laravel) |
| API | Postman / Laravel HTTP Test |
| Giao diện web | Kiểm thử thủ công trên trình duyệt Chrome |
| Thiết bị Pi | Kiểm thử thực tế trên phần cứng |
| Dữ liệu mẫu | Laravel Seeder + Factory (Faker) |

**Môi trường test:** Local development (Laravel Sail / XAMPP), DB test riêng biệt không ảnh hưởng dữ liệu production.
